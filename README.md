# desafio-ec2-aws
# Gerenciando Instâncias EC2 na AWS

## 📌 Descrição

Este projeto foi desenvolvido como parte do desafio da DIO com foco no gerenciamento de instâncias EC2 na AWS.

O objetivo foi praticar conceitos importantes relacionados à criação de máquinas virtuais na nuvem, utilização de imagens AMI e gerenciamento de snapshots EBS.

---

# ☁️ O que foi aprendido

Durante o laboratório foram abordados os seguintes conceitos:

- Criação e gerenciamento de instâncias EC2
- Tipos de instâncias na AWS
- Configuração de segurança
- Criação de imagens AMI
- Snapshots de volumes EBS
- Boas práticas de gerenciamento na AWS

---

# 🚀 Amazon EC2

O Amazon EC2 (Elastic Compute Cloud) é um serviço da AWS que permite criar servidores virtuais na nuvem.

Com ele é possível:

- Hospedar aplicações
- Criar servidores Linux ou Windows
- Escalar recursos conforme necessidade
- Gerenciar acesso e segurança

---

# 🖼️ AMI (Amazon Machine Image)

As AMIs são imagens utilizadas para criar instâncias EC2.

Elas armazenam:
- Sistema operacional
- Configurações
- Aplicações instaladas
- Dados necessários para inicialização

## Vantagens
- Facilidade na replicação de ambientes
- Backup de configurações
- Agilidade na criação de servidores

---

# 💾 Snapshots EBS

Os Snapshots EBS são backups dos volumes utilizados nas instâncias EC2.

## Principais benefícios
- Backup de dados
- Recuperação de informações
- Criação de novos volumes
- Segurança contra perda de dados

---

# 📚 Conclusão

Este desafio permitiu consolidar conhecimentos sobre gerenciamento de instâncias EC2 na AWS, além de compreender melhor a utilização de AMIs e Snapshots EBS no ambiente de computação em nuvem.
